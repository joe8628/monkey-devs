# Stack Decision

Make the final, binding technology stack decision. This is irreversible within the current workflow.

Required output:
1. **Chosen Stack**: name every framework, language, database, and tool
2. **Final Rationale**: why this stack over the evaluated alternatives
3. **Rejected Alternatives**: one sentence per rejected option explaining the deciding factor
4. **Stack Record**: write the decision into `docs/architecture.md` under a "## Technology Stack" section

After this stage, no stack discussion or reconsideration is in scope. All subsequent stages use the chosen stack.
