# README Writing

Write `README.md` with these sections in order:

1. **Project name** (H1 heading)
2. **One-sentence description** of what the project does
3. **Prerequisites**: runtime versions, required env vars
4. **Installation**: exact commands to install and set up
5. **Usage**: 2–3 concrete examples with expected output
6. **Running Tests**: exact test command
7. **Project Structure**: brief directory tree with one-line descriptions

Use code blocks for all commands. Do not use marketing language. Be direct and specific.
