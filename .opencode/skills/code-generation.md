# Code Generation

Write production-quality code that:
- Matches the naming conventions and style of existing files in the project
- Handles errors only at system boundaries (user input, external APIs, filesystem)
- Contains no TODO comments or placeholder logic
- Is the simplest implementation that satisfies the task description

Do not add features beyond what the task specifies. Do not add comments explaining what the code does — write self-documenting code instead. Add comments only where the logic is genuinely non-obvious.
