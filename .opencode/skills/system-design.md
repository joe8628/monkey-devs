# System Design

Produce a complete system design covering:

1. **Component Diagram** (ASCII art): show all major components and their relationships
2. **Data Flow**: describe how data moves between components
3. **Component Responsibilities**: one paragraph per component
4. **Interface Contracts**: public APIs, CLI commands, or inter-process communication
5. **Data Models**: key entities and their fields

Be specific. Name every component. Reference the requirements spec when explaining design decisions.
