# Developer Guide Writing

Produce `docs/developer-guide.md` covering:

1. **Project Structure**: annotated directory tree
2. **Module Descriptions**: purpose of each top-level module
3. **Environment Setup**: exact steps to create a dev environment
4. **Running Tests**: command and expected output
5. **Adding a New Skill**: step-by-step: write the markdown file, add to registry.yaml
6. **Adding a New Tool**: step-by-step: implement in tools.py, add to registry.yaml, update get_stage_tools()

Write for a developer who is new to this codebase but experienced with Python.
